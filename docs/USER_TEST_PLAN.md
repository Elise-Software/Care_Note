# 실사용자 모집 전 사용자 테스트 계획서

## 목적

고령 부모를 둔 보호자와 일반 사용자가 텍스트·사진·음성으로 진료 후 내용을 입력하고, 가족 체크보드와 공유문을 이해·활용할 수 있는지 확인한다.

## 대상

- 고령 부모를 둔 비동거 가족 보호자
- 병원 안내문을 가족에게 전달해 본 경험이 있는 일반 사용자

## 진행 원칙

- 실제 환자 정보, 실제 병원 문서, 실제 음성은 사용하지 않는다.
- 합성 fixture와 참가자가 직접 작성한 비식별 예시만 사용한다.
- 서비스가 의료 진단이나 처방을 제공하지 않는다는 점을 먼저 안내한다.
- 전사문·사진 추출문은 사실 확인이 필요한 초안임을 설명한다.

## 과업

1. 직접 입력 탭에서 가상 진료 후 메모를 입력하고 분석한다.
2. 사진 탭에서 합성 안내문을 업로드하고 추출문을 수정·확정한다.
3. 음성 탭에서 짧은 가상 음성을 녹음하거나 fixture를 업로드하고 전사문을 수정·확정한다.
4. 서로 다른 일정이 포함된 두 자료를 분석해 `확인 필요` 경고를 찾는다.
5. 체크보드에서 완료와 담당자를 지정한다.
6. 공유문에서 완료 항목 포함 여부를 바꾸고 복사 또는 다운로드한다.

## 평가 항목

- 서비스 목적과 의료 안전성 문구 이해도
- 직접 입력·사진·음성 탭의 발견 가능성
- 사진 추출문/전사문을 검토해야 한다는 점의 이해
- 개인정보 유사 패턴 경고의 이해와 부담감
- 충돌 경고와 원문 근거의 이해
- 체크보드와 담당자 지정의 유용성
- 공유문의 전달 편의성
- 반복 사용 의향과 개선 요구사항

## 측정 방법

- 과업 성공/실패와 소요 시간
- 5점 리커트: 입력 편의성, 결과 이해도, 체크보드 유용성, 공유문 유용성, 신뢰감
- 관찰 메모: 막힌 단계, 오해한 문구, 원하는 수정

## 현재 검증 상태

실제 10명 테스트를 완료했다고 기록하지 않는다. 현재 단계는 합성 fixture 기반 내부 기능 검토, 결정론적 20개 평가, 브라우저 smoke/E2E 캡처까지 완료된 상태다. 실제 모집 후 결과는 이 계획서와 별도 보고서로 구분한다.
