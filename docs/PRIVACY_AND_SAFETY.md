# 개인정보 및 안전성

## 개인정보 처리 원칙

- 서비스는 실제 개인정보 입력을 요구하지 않는다.
- 전화번호, 일반 전화, 이메일, 주민등록번호 유사값, 긴 숫자 ID를 브라우저에서 가능성 패턴으로 경고한다.
- 경고는 확정 판정이 아니며, 예: `010-0000-0000 → 010-****-0000`처럼 마스킹해 표시한다.
- 원문과 원본 파일은 앱 로그나 평가 artifact에 기록하지 않는다.
- 사진·음성 AI 분석을 선택한 경우에만 선택 자료가 private SSH tunnel을 통해 일회성 gateway에 전송된다.
- gateway는 업로드 오디오의 임시 파일을 요청 종료 후 삭제하고, 원본 이미지를 저장하지 않는다.

## API 경계

브라우저는 클라우드 IP나 SSH 키를 직접 사용하지 않는다.

```text
Browser → localhost WSL BFF → SSH local forward → 127.0.0.1 cloud gateway → Gemma 4 / STT
```

WSL에서 Windows 파일 시스템에 있는 개인키는 권한이 넓게 보이므로 `scripts/start.sh`가 실행 중에만 `/tmp`의 0600 임시 복사본을 만든다. 이 파일은 서버 종료 시 삭제된다. 저장소에는 키나 `.env`를 넣지 않는다.

## 의료 안전성

모델과 규칙 엔진은 입력에 없는 의료 지시를 생성하지 않는다.

- 새 약 이름, 복용량, 진단명, 응급 판정을 추가하지 않는다.
- `sourceText`가 입력에 없거나 날짜·시간이 원문과 맞지 않는 AI 항목은 제거한다.
- 사진·음성 추출문은 사용자가 확인하기 전에는 분석 source가 아니다.
- source 간 일정이 다르면 `확인 필요`로 표시한다.

UI 고지: “본 서비스는 의료적 진단이나 처방을 제공하지 않으며, 사용자가 입력하거나 제공한 자료를 정리하고 공유하기 위한 정보지원 도구입니다.”
